🧩 How it connects — shared backend = dynamic, live in production

🏷️ Seller apps

Separate web (Next.js) + mobile (Expo). Own UI & seller auth.

→submit + docs

🗄️ Shared Supabase

business_submissions (pending) → Verification Console → on approve, upsert into business_listings (status: Active).

→live read

💻 Main buyer app

Reads business_listings in real time. Approved listing appears instantly — no redeploy.

"Will it be dynamic / live in production?"  Yes. Because the seller app and the main app share the same Supabase database, publishing is a data change, not a code change. The moment your team flips a submission to approved, the row lands in business_listings with status = 'Active' — and the production buyer app, which queries that table live (even via Supabase realtime), shows it to buyers instantly. No rebuild, no deploy, no waiting. The two apps stay separate products; they just meet at one shared listings table.
