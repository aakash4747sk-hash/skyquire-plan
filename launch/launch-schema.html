Build sequence the data follows: P1 adds the marketplace money + AI layer (pgvector embeddings, health_scores, Razorpay transactions, S3 storage). P2 layers the corporate side (valuations, dd_checklists, subscriptions, team workspaces). P3 is the trust layer (eSign envelopes, escrow, advisors, multi-channel notifications). P4 brings mobile + analytics + a security audit. Everything is RLS-scoped from day one using the can_access_deal() pattern already live. Buy-don't-build for eSign, escrow, and payments — integrate regulated vendors, never custom. A separate Seller Platform (own web + mobile) writes to new business_submissions tables; on team approval the listing upserts into business_listings with status Active and appears live to buyers — shared backend, no redeploy.
