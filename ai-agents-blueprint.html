AI Agents Blueprint

Skyquire is a mediated marketplace — a human sits in the middle of every listing, conversation, and deal. That's what makes it trustworthy, and what won't scale. This blueprint turns "AI features" into an agentic operations layer: a team of specialized agents that run the back office so one founder operates like a 10-person M&A firm — with you approving anything that matters.

② The agent team — what each does & how much it's trusted

🔭 Scout Draft

Sourcing — attacks the #1 risk: supply.
  • ▹ Finds likely-to-sell businesses from permissioned public sources
  • ▹ Drafts personalized outreach (owner / CA / broker)
  • ▹ Qualifies + scores inbound seller leads
  • ▹ Tracks the CA-partner pipeline & follow-ups
Tools: web search · Supabase · email/WhatsApp drafts · the Supply plan's scripts

🛡️ Verifier Suggest

First-pass listing verification.
  • ▹ Parses uploaded docs (P&L, balance sheet, GST, MCA)
  • ▹ Validates GSTIN / CIN; cross-checks the numbers
  • ▹ Pre-fills your verification checklist + flags red flags
  • ▹ You approve — never auto-verifies
Tools: doc parsing · Supabase storage · verification_checkpoints · (extends the DD assistant)

✍️ Curator Draft

Raw submission → polished, anonymized teaser.
  • ▹ Writes description, highlights, sector tags
  • ▹ Generates the embedding (you already auto-embed)
  • ▹ Asking-price sanity check vs sector comps
  • ▹ Strips identifying details (confidentiality)
Tools: Anthropic · pgvector · business_listings

✦ Aria+ Act (alerts)

Proactive matchmaking — already half-built.
  • ▹ On a new listing, finds the buyers/mandates it fits
  • ▹ Queues match alerts (email/in-app)
  • ▹ Explains *why* it's a fit (scored, like today)
Tools: pgvector match_listings · buyer_profiles · saved_searches · notifications

📨 Relay Draft

Mediation — the highest-leverage agent.
  • ▹ Drafts the buyer↔team↔seller relay messages
  • ▹ Auto-answers common buyer questions from listing data
  • ▹ Triages inquiries by seriousness; summarizes threads
  • ▹ You hit send — confidentiality wall enforced
Tools: messages · inquiries · listing context · Anthropic

🩺 Diligence Suggest

DD assistant, grown up (post-launch).
  • ▹ Ingests a data room → structured DD report
  • ▹ Generates the buyer's diligence question list
  • ▹ Risk & anomaly flags, health score
Tools: deal_room_documents · doc parsing · /api/ai/diligence

🧭 Coordinator Draft

Deal progression (post-launch).
  • ▹ Moves deals through stages; nudges stalled ones
  • ▹ Drafts NDAs / LOIs from templates
  • ▹ Status summaries for both sides
Tools: deals · deal_tasks · scheduled tasks · email

📣 Marketer Draft

Feeds the content engine.
  • ▹ Drafts daily posts from the content calendar
  • ▹ Repurposes one idea into many formats
  • ▹ Drafts waitlist nurture emails
Tools: content calendar · waitlist · Anthropic
③ Architecture — how the team fits together

🧱 The shape

  • → An orchestrator routes work to specialist sub-agents (each a focused prompt + tool set).
  • → Event-triggered agents fire on a submission / inquiry / new listing.
  • → Proactive agents (Scout, Aria+) run as scheduled tasks.
  • → Output lands as drafts & suggestions in the admin console — your existing approval surface.
  • → Built on what exists: Anthropic API in routes, pgvector, the DD assistant, auto-embed.

🧰 Shared tools (the agent toolbox)

  • → Supabase (RLS-scoped reads/writes) — listings, inquiries, messages, deals
  • → pgvector semantic search / matching
  • → Document parsing (financials, GST, MCA PDFs)
  • → Web search (sourcing, comps)
  • → Messaging (email via Resend, WhatsApp later) — draft-only by default
④ Guardrails — non-negotiable
1. Human-approval gate on every side-effectful action
Sending a message, publishing a listing, charging a fee, moving money — an agent drafts; a human commits. Agents never auto-send to a buyer/seller or auto-verify.
2. Confidentiality wall
Agents must never leak buyer ↔ seller identity across the mediated channel. The Relay agent works within the same channel isolation your RLS already enforces.
3. Compliance line — no investment advice
Agents present information & tools, never "advice" or return promises. This keeps the product on the right side of the SEBI/adviser question (see Legal Critical Path) and must be in every agent's system prompt.

🎯 Recommended first build: the Relay agent. Mediation is your core scaling limit — every extra conversation today costs your time. A draft-and-approve relay assistant in the admin console gives the biggest leverage for the least risk, and proves the agentic pattern you'll reuse for the rest.

Beyond operations, this is a story. Your landing page already says "powered by AI" — an agentic ops layer makes Skyquire a genuinely AI-native M&A platform, not a directory with a chatbot. That's a sharper wedge for press, partners, and investors: "one founder running an M&A firm with an AI back office." Build it for leverage; it pays off twice as narrative.
